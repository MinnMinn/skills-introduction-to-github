# Contributing to Skills — Introduction to GitHub

Thank you for taking the time to contribute! 🎉  
This document explains the conventions to follow so that the review process stays smooth and consistent.

---

## How to Contribute

1. **Fork** the repository and clone your fork locally.
2. Create a **new branch** from `main` with a descriptive name:
   ```bash
   git checkout -b feat/short-description
   ```
3. Make your changes, following the conventions below.
4. **Commit** your changes with a clear message (see [Commit Messages](#commit-messages)).
5. **Push** your branch and open a **Pull Request** against `main`.
6. A maintainer will review your PR. Please respond to feedback promptly.

---

## Commit Messages

Use the [Conventional Commits](https://www.conventionalcommits.org/) style:

```
<type>: <short summary>
```

Common types:

| Type | When to use |
|------|-------------|
| `feat` | New feature or content |
| `fix` | Bug fix |
| `docs` | Documentation only |
| `chore` | Maintenance (deps, config) |
| `refactor` | Code change that neither fixes a bug nor adds a feature |

Example:
```
docs: add workflow diagram to README
```

---

## Pull Request Checklist

Before submitting your PR, please confirm:

- [ ] My branch is up to date with `main`
- [ ] My commit messages follow the Conventional Commits style
- [ ] I have updated relevant documentation if needed
- [ ] My changes do not introduce broken links or typos

---

## Code of Conduct

Be respectful and constructive. This is a learning environment — everyone is here to improve.
